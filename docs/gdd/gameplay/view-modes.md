# View Modes & Optimizations

AeroBeat still separates gameplay logic from runtime presentation, but current product wording should favor the PC-first camera slice.

## Core views

### Portal View

- immersive 3D presentation
- still useful as a runtime/view concept
- relevant for future VR return and larger-screen experiences

### Track View

- linear 2D presentation family
- especially important for compact or straightforward PC camera-friendly layouts

## Scope guidance

The docs should avoid implying that VR and mobile view parity are already equal-status shipping targets.

- PC camera presentation is the main current priority
- mobile-oriented adaptation remains follow-on work
- VR-oriented presentation remains a later platform slice

## Runtime adaptation

Renderer adaptation across views still matters, especially for preserving authored choreography across different presentation surfaces. That is a technical capability, not a promise of immediate platform parity.
